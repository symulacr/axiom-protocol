  THE  9-STEP  E2E  (the canonical flow, 9/9 live)
================================================================================

  ┌────┬─────────────────────────────────────────────────────────────────┐
  │  1 │ GET /health                                                    │
  │    │ F → B  →  { ok, chainHead: 38,919,827 }                        │
  ├────┼─────────────────────────────────────────────────────────────────┤
  │  2 │ Build StrategySpec (targetToken, threshold, action)             │
  │    │ client-side JSON only                                          │
  ├────┼─────────────────────────────────────────────────────────────────┤
  │  3 │ AES-256-GCM encrypt(StrategySpec)                              │
  │    │ ECIES.seal(dataKey, deployer.pub)                              │
  │    │ real crypto: Node crypto + eciesjs; no mocks                   │
  ├────┼─────────────────────────────────────────────────────────────────┤
  │  4 │ upload(blob + key)  →  0G Storage                              │
  │    │ { rootHash: 0xcd77…543f, tx: 0x…e7ba2 }                        │
  ├────┼─────────────────────────────────────────────────────────────────┤
  │  5 │ POST /v1/agents/mint                                           │
  │    │ B  →  200 { ok, dataHash, txHash }                             │
  ├────┼─────────────────────────────────────────────────────────────────┤
  │  6 │ POST /v1/vaults/0/deposit  (0.1 OG)                            │
  │    │ B → Vault  →  deposit(tokenId)  →  deposit event               │
  ├────┼─────────────────────────────────────────────────────────────────┤
  │  7 │ POST /v1/vaults/0/strategy  (merkleRoot, dailyLimit)            │
  │    │ B → Vault  →  setStrategy(tokenId, merkleRoot, dailyLimit)     │
  ├────┼─────────────────────────────────────────────────────────────────┤
  │  8 │ POST /v1/orchestrator/tick    (fan-out)                         │
  │    │  ├─ Storage.downloadToBlob(rootHash) → bytes                   │
  │    │  ├─ Compute.chat(messages, model)  (per-call processResponse)  │
  │    │  └─ Vault.read onchain state                                   │
  │    │  →  { recommendation, rawModelOutput, onchain, storage, …}     │
  ├────┼─────────────────────────────────────────────────────────────────┤
  │  9 │ POST /v1/agents/0/transfer  (to, receiverPubKey64, dataHash)    │
  │    │  a) B → Oracle: POST /v1/ownership                             │
  │    │        { dataHash, sealedKey, targetPubkey, nonce, validUntil } │
  │    │  b) Oracle: EIP-191 sign with registeredSigner key              │
  │    │        → { signature, signer: 0x4373…F91 }                      │
  │    │  c) B → AxiomAgentNFT.iTransferFrom(from, to, tid,             │
  │    │                                 ownershipProof, accessProof)   │
  │    │  d) NFT → Verifier.verifyTransferValidity(proof[])             │
  │    │        (EIP-712 validUntil within 7d window)                    │
  │    │  e) NFT  →  Transfer event                                     │
  └────┴─────────────────────────────────────────────────────────────────┘

  4 external functions:
    1. mint(to, dataHash, sealedKey)              onlyOwner

    3. authorizeUsage(tid, user, expiresAt)       onlyOwner or delegated
