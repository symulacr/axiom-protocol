
# Changelog — v0.2.6 Server Hardening

---

## Server Cleanup

- **`loadEnv` centralized**: `server.ts` no longer calls `loadEnv()` — single call at `index.ts` module scope. Re-exported from `@axiom/config/env`.
- **Error boundary deduped**: Single `app.use` error handler replaces scattered per-route catches.
- **Request ID tracing**: `x-request-id` header on all responses for log correlation.

# Changelog — v0.2.5 Codegen + Route Registry + Dead Code Cleanup

---

## wagmi CLI Codegen

- ABIs now generated by `@wagmi/cli` from Forge `out/` JSON artifacts (`wagmi generate`).
- Generated types in `@axiom/config/abis/generated.ts` (`axiomAgentNftAbi`, `axiomStrategyVaultAbi`, etc.).
- Hand-written ABI exports preserved for backward compatibility.

## Route Registry

- `REGISTERED_ROUTES` export in `route-factory.ts` registers every `createRoute()` call.
- Exposed at `GET /v1/admin/routes` (API-key gated by global `createApiKeyAuth` middleware).
- Each route carries `consumer` and `description` fields for frontend discovery.

## Dead Routes Deleted

- `protocol/stats`, vault-specific endpoints, `compute/pay`, and 7 other legacy routes removed from `server.ts`.
- `route-schemas.ts` pruned: unused schemas removed.

## EIP-712 Domain Fix

- Domain name changed from `"Axiom iNFT"` → `"AxiomTeeVerifier"` in oracle signer (`apps/oracle/src/crypto/eip712.ts`).
- Frontend `useEip712Domain` hook updated to use `getAxiomTeeVerifierAddress(chainId)` and matching domain name.
- Matches on-chain `AxiomTeeVerifier._domainSeparator()`.

## Frontend Polling on react-query

- New `usePolledApi` hook wraps `@tanstack/react-query` `useQuery` — accepts static URL or getter function with explicit `queryKey`.
- `useProviders.ts`: migrated to `usePolledApi` (30s poll).
- `useEventHistory.ts`: migrated to cursor-based incremental polling via `usePolledApi`.
- Legacy `usePoll.ts` marked deprecated but retained for non-HTTP fetchers.

## Cross-Cutting

- `packages/config/wagmi.config.ts` declares 5 contracts from `packages/config/abis/*.json`.
- `@tanstack/react-query` added to frontend deps. `QueryClientProvider` wraps app in `main.tsx`.
- Frontend ABIs now re-export from `@axiom/config/abis` (canonical source).

# Changelog — v0.2.4 DA gRPC Removal

---

## Deleted

- Vendored gRPC client (`da-client.ts`, `da.ts`, `disperser.proto`) deleted from indexer source. Only stale `dist/` artifacts remain (unreferenced).
- `DA_GRPC_URL`, `DA_GRPC_CA_CERT`, `DA_GRPC_TLS_ENABLED` env vars no longer read.

## Changed

- Indexer event persistence migrated to 0G Storage exclusively: batched upload via `@0gfoundation/0g-storage-ts-sdk` `Indexer` class.
- `sink.ts` simplified: events POST to backend + batched 0G Storage upload (gRPC path removed).
- `env.ts` re-exported from `@axiom/config/env` — single source of truth.
- Docker Compose `da-client` sidecar service retained but vestigial (indexer does not depend on it).

# Changelog — v0.2.3 Compute Router Migration

---

## Compute Router (OpenAI-compatible)

- `createRouterClient()` now returns an `OpenAI` instance from the `openai` npm package.
- Router base URL resolved via `getComputeBaseUrl()`: env var → chain config → Galileo fallback.
- Direct SDK proxy support via `AXIOM_COMPUTE_DIRECT_KEY` (app-sk-* tokens).
- Fallback to `AXIOM_COMPUTE_API_KEY` / `OG_COMPUTE_API_KEY` for Router API auth.
- No hardcoded fallback URLs.

## Provider Discovery

- Custom `InferenceServing.getAllServices()` RPC wrapper replaced with 0G Compute SDK's `ReadOnlyInferenceBroker.listService()`.
- Result cached for process lifetime (mutex-guarded lazy init).
- `resolveProviderUrl()` resolves inference endpoints from on-chain registry.

## Streaming

- SSE streaming via `stream: true` + async iterator from OpenAI SDK.
- Removed fake word-splitting post-processing (was producing broken token boundaries).
- WSS streaming tick: tokens burst to websocket subscribers after inference completes.
